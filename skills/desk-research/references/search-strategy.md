# 調査スコープ設計と検索クエリ戦略

## 1. 調査スコープの設計

### 1.1 論点から調査項目への分解

提案書の各論点（大論点・小論点）を、具体的な調査項目に分解する。分解の粒度は「1つの検索セッションで答えが出せる単位」を目安とする。

**分解の手順**:

1. 大論点を読み、「この問いに答えるために必要な情報は何か」をリストアップ
2. 小論点ごとに「何が分かれば仮説を検証できるか」を定義
3. 各仮説に対して「支持する根拠」「否定する根拠」それぞれの情報源を想定
4. 情報の種類を分類:
   - **定量データ**: 市場規模、年収、価格、シェア、成長率など数値で表せる情報
   - **定性情報**: 業界動向、意思決定要因、戦略、課題など文脈で理解する情報
   - **事例・ケース**: 具体的な企業事例、成功/失敗事例、ベストプラクティス

**分解の例**:

```
大論点: 「○○市場の年収相場はどの水準か？」
  ├── 調査項目A: 転職エージェント実績データの収集 [定量]
  ├── 調査項目B: 求人票に記載された年収レンジの網羅的収集 [定量]
  ├── 調査項目C: 口コミサイトの年収投稿データ [定量+定性]
  ├── 調査項目D: 公的統計・業界団体レポート [定量]
  └── 調査項目E: フリーランス市場との比較 [定量]
```

### 1.2 網羅性チェック: 論点マトリクス

調査項目を洗い出した後、以下のマトリクスで網羅性を検証する。

| 論点 | 必要な情報の種類 | 調査項目 | 想定ソース | 優先度 |
|------|----------------|---------|-----------|--------|
| 大論点① | 定量 | 項目A | ソースa, b | 高 |
| └ 小論点①-1 | 定量+定性 | 項目B, C | ソースc, d | 高 |
| └ 小論点①-2 | 定性+事例 | 項目D | ソースe | 中 |
| 大論点② | ... | ... | ... | ... |

**チェック観点**:

- 各大論点に最低2つ以上の調査項目が紐づいているか
- 各小論点の仮説を検証できる情報源が具体的に想定されているか
- 定量と定性の両面から検証できる構成になっているか
- 「仮説を否定する情報」を探す項目も含まれているか（確証バイアスの回避）

### 1.3 優先順位の設定

調査項目の優先順位を以下の基準で設定:

- **高**: クライアントの意思決定に直結する / 提案書の核心的な仮説に関わる
- **中**: 論点への回答を補強する / 比較・裏付けに必要
- **低**: あれば望ましいが、なくても結論は出せる

時間制約がある場合は高→中の順に実行し、低は省略可能とする。

## 2. 検索クエリの設計

### 2.1 クエリ設計の基本方針

1つの調査項目に対し、**最低5つの検索クエリ**を異なる切り口で設計する。同じ情報を異なるクエリで探すことで、単一ソースへの依存を避け、情報の多角的な裏取りを実現する。

### 2.2 切り口の展開方法

一つのトピックに対して、以下の6軸でクエリを展開する:

#### A. 語彙の展開

| 展開タイプ | 説明 | 例（テーマ: ERP導入PM年収） |
|-----------|------|---------------------------|
| 類義語 | 同じ意味の別表現 | 「年収」→ 給与, 報酬, 賃金, 処遇 |
| 上位概念 | より広いカテゴリ | 「ERP導入PM」→ ITコンサルタント, PM全般 |
| 下位概念 | より狭い特定対象 | 「ERP導入PM」→ SAP導入PM, Oracle ERP PM |
| 業界固有用語 | その業界でのみ通じる表現 | 「PM」→ プロジェクトマネージャー, PjM, PMO |
| 英語表現 | 英語での検索 | "ERP implementation project manager salary Japan" |
| 略称・正式名称 | 略と正式の両方 | SAP, SAP S/4HANA, Systems Applications and Products |

#### B. ソース種類別のクエリ

| ソース種類 | クエリの特徴 | 付加キーワード例 |
|-----------|-------------|----------------|
| 公的統計・政府レポート | 信頼性最高。政策文書を狙う | site:go.jp, 白書, 統計, 調査報告書 |
| 業界団体・協会 | 業界固有データ | 協会, 連合会, 業界レポート, 年次報告 |
| 調査会社レポート | 市場規模・予測データ | IDC, Gartner, 矢野経済研究所, 市場規模 |
| 上場企業IR | 財務・事業データ | 有価証券報告書, IR, 決算説明資料, 統合報告書 |
| 転職系メディア | 年収・求人データ | 転職, 求人, 年収, キャリア |
| ニュース・メディア | 最新動向・トレンド | 日経, 2025, 2026, 最新, 動向 |
| 学術・シンクタンク | 分析・研究 | 論文, 研究所, レポート, 分析 |

#### C. 時間軸の展開

- **最新データ**: 「2025 2026」「最新」「直近」を付与
- **推移・トレンド**: 「推移」「変化」「トレンド」「年次比較」を付与
- **予測**: 「予測」「見通し」「forecast」を付与

### 2.3 クエリテンプレート

調査項目1つに対して、以下のテンプレートで5本以上のクエリを生成する:

```
クエリ1: [対象] [知りたい情報] [時期] （基本クエリ）
クエリ2: [類義語/下位概念] [知りたい情報] [信頼性の高いソースキーワード]
クエリ3: [上位概念] [知りたい情報] [比較・ベンチマーク]
クエリ4: [英語表現] [知りたい情報] Japan [年]
クエリ5: [対象] [知りたい情報] [否定・反証方向のキーワード]
クエリ6+: site:指定 や filetype:pdf など絞り込みクエリ
```

**具体例（調査項目: ERP PM/PLの決定年収レンジ）**:

```
1. "ERP PM 年収 転職 2025 2026"
2. "SAP導入 プロジェクトマネージャー 給与 転職実績"
3. "ITコンサルタント PM 年収相場 比較"
4. "ERP implementation project manager salary Japan 2025"
5. "ERP PM 年収 下落 低下 減少"（反証方向）
6. "ERP コンサルタント 年収" site:jac-recruitment.jp
7. "プロジェクトマネージャー 年収" filetype:pdf
```

### 2.4 検索実行のルール

- **並列実行**: 独立した調査項目のクエリは可能な限り並列にWebSearchを実行
- **深掘り判断**: WebSearchの結果から有望なURLが見つかったら、WebFetchで全文取得
- **反復**: 1回の検索で十分な情報が得られない場合、クエリを修正して再検索
- **ソースの多様性**: 1つの調査項目につき最低3つの異なるソースから情報を取得
- **情報の鮮度**: 可能な限り直近1-2年のデータを優先。古いデータしかない場合はその旨を明記

### 2.5 検索が不十分な場合の判断基準

以下のいずれかに該当する場合、Layer 2（Browser Use）またはLayer 3（Deep Research）への移行を検討:

- 3回以上クエリを変えても有用な情報が得られない
- ログインが必要なサイトにしか情報がない
- 動的ページの操作（フィルタ、ソート等）が必要
- 非公開情報や有料レポートの内容が必要
- 定量データが十分に集まらない（裏取り用の3ソースが揃わない）

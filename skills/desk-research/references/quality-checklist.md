# 調査レポート品質チェックリスト

レポート完成後、以下の全項目を確認する。

---

## 1. 網羅性

- [ ] 提案書の全ての大論点に対応するセクションがあるか
- [ ] 各小論点に対して「主張」「根拠」「ソース」が揃っているか
- [ ] 仮説検証シートで全ての初期仮説の判定（支持/否定/修正/未検証）が記載されているか
- [ ] 未検証の仮説がある場合、補完方法が明記されているか

## 2. 根拠の質

- [ ] 各主張に最低1つの具体的ソースが紐づいているか
- [ ] 定量データには出典元・サンプル数・データ期間が明記されているか
- [ ] 1つの調査項目に対して複数ソースからの裏取りがあるか（理想は3ソース以上）
- [ ] ソースの信頼性評価（高/中/低）が記載されているか
- [ ] 口コミ・自己申告ベースのデータにはその旨の注記があるか

## 3. 正確性

- [ ] 元のソースにない数字・固有名詞・事例を捏造していないか
- [ ] 引用した数値とソースの記載が一致しているか
- [ ] 推定値・概算値にはその旨が明記されているか（「推定」「概ね」「約」等）
- [ ] 出典URLが実在し、アクセス可能か

## 4. 論理性

- [ ] 主張と根拠の間に論理的飛躍がないか
- [ ] 確証バイアスに偏っていないか（仮説を否定する情報も探したか）
- [ ] 相関と因果を混同していないか
- [ ] 一般論で終わらず、具体的な示唆に落とし込んでいるか

## 5. 仮説検証の質

- [ ] 各仮説の判定理由が具体的に記載されているか
- [ ] 更新後の仮説は根拠に基づいた修正になっているか
- [ ] 仮説の支持/否定の判断が、単一ソースだけに依存していないか
- [ ] 新たに発見した事実から追加仮説を提示しているか（該当する場合）

## 6. 表現・記述

- [ ] 「把握する」「整理する」「深掘りする」だけで終わる文がないか
- [ ] 「一般的に」「多くの場合」「状況によって異なります」等の安全クッションが不要に入っていないか
- [ ] 同義語の言い換え連打（重要・大切・欠かせない等）がないか
- [ ] 「」の多用がないか
- [ ] ()の多用がないか（補足は本文に組み込む）
- [ ] 表は比較・整理の目的で使われているか（装飾的な表は不要）

## 7. 構成・読みやすさ

- [ ] 結論が冒頭にあるか（先に概要、後に詳細）
- [ ] 大論点ごとのセクション分離が明確か
- [ ] 参考文献リストが番号順に整理されているか
- [ ] 本文中の参照番号 `[N]` と参考文献リストが対応しているか

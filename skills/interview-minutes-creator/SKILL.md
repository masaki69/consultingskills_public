# Interview Minutes Creator

インタビューの質問リストと文字起こしから、構造化された詳細な議事録を作成する。

## ワークフロー

1. **入力ファイルの確認**
   - 質問リスト（質問項目と意図が記載されたファイル）
   - インタビュー文字起こし（音声認識等で生成されたテキスト）

2. **内容の分析**
   - 質問リストの各質問と対応する回答を文字起こしから特定
   - インタビュイーのプロフィール情報を抽出
   - 重要なファクト（数値、固有名詞、具体例）を抽出

3. **議事録の構成**
   - 出力フォーマットに従って議事録を作成
   - 各質問への回答を第一階層（核心文）と第二階層（詳細）に整理

4. **品質チェック**
   - 全ての質問に回答が紐づいているか確認
   - 専門用語に説明が付与されているか確認
   - 第一階層の文章で内容の全体像が把握できるか確認

## 出力フォーマット

詳細なフォーマットガイドは `references/output-format.md` を参照。

### 基本構成

```markdown
# [インタビュータイトル]

## インタビュー概要
| 項目 | 内容 |
|------|------|
| 実施日時 | YYYY年MM月DD日（曜日）HH:MM〜HH:MM |
| 所要時間 | 約XX分 |
| 実施形式 | オンライン/対面 |

## インタビュイープロフィール
[氏名、所属、役割、専門性]

## ファクトサマリ
[重要なポイントを端的に箇条書き]

## インタビュー内容
### 質問N：[質問文をそのまま記載]
- **[核となる情報を端的に表現した完全な文]**
    - [詳細1]
    - [詳細2]
    - ...

## ヒアリングの詳細（逐語録）
[重要な発言を引用形式で記載]

## まとめ
[全体の総括]
```

## 記述ルール

### 第一階層（太字の核心文）
- 質問への回答を1-2文で端的に表現
- 必ず完結した文章で、単文で意味が理解できること
- 太字（`**`）で強調

### 第二階層（箇条書きの詳細）
- 第一階層を補足する具体的な情報
- 技術・概念の説明、具体例、背景情報、課題などを含む
- 体言止めは極力使わない

### 用語の扱い
- 専門用語は初出時に（）内で簡潔に説明
- 略語は初出時にフルスペルと説明を記載
- 業界特有の概念は具体例を交えて説明

### 品質基準
- 質問文は原文のまま使用
- インタビュー内容を可能な限り盛り込む
- 単文で意味が理解できる完結した文章
- 重要なポイントは太字で強調

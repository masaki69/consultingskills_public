# 議事録出力フォーマット詳細ガイド

## 全体構成

議事録は以下の6つのセクションで構成する。

### 1. インタビュー概要

テーブル形式で基本情報を記載。

```markdown
## インタビュー概要

| 項目 | 内容 |
|------|------|
| 実施日時 | 2026年1月23日（金）10:00〜10:53 |
| 所要時間 | 約53分 |
| 実施形式 | オンライン |
```

### 2. インタビュイープロフィール

```markdown
## インタビュイープロフィール

**氏名：山田 太郎（ヤマダ タロウ）**

### 所属・役割
株式会社〇〇 / △△部 / 営業担当

### 専門性
[具体的な専門領域や強みを記載]
```

### 3. ファクトサマリ

インタビュー全体から抽出した重要なポイントを、カテゴリ別に整理。

```markdown
## ファクトサマリ

### [カテゴリ名1]
- **[重要ポイント]**：[詳細説明]
- [補足情報]

### [カテゴリ名2]
- **[重要ポイント]**：[詳細説明]
```

### 4. インタビュー内容

各質問に対する回答を、二階層構造で記載。

#### 第一階層の書き方

- 質問への回答の核心を1-2文で表現
- 必ず**太字**で強調
- 完結した文章で、この文だけで回答の要点が理解できること

**良い例：**
```markdown
- **小ロット対応が最大の強みであり、1台から100台程度の製作に対応できる点が競合との差別化ポイントとなっている**
```

**悪い例：**
```markdown
- **小ロット対応が強み**  ← 体言止め、情報量不足
- **小ロットでの対応について**  ← 内容が不明確
```

#### 第二階層の書き方

- 第一階層を補足する詳細情報
- 具体例、数値、背景説明、課題などを含む
- 体言止めは使わず、完結した文で記載

```markdown
### 質問1：[質問文をそのまま記載]

- **[第一階層：核心文]**
    - [詳細1：具体的な説明や事例]
    - [詳細2：背景情報や理由]
    - [詳細3：数値データや実績]
    - [詳細4：課題や注意点]
```

### 5. ヒアリングの詳細（逐語録）

特に重要な発言を引用形式で記載。テーマごとにグループ化。

```markdown
## ヒアリングの詳細（主要部分の逐語録）

### [テーマ名]

> **[発言者名]**：「[発言内容をそのまま記載]」

> **[発言者名]**：「[発言内容]」
```

### 6. まとめ

インタビュー全体から得られた知見を総括。

```markdown
## まとめ

本インタビューを通じて、以下のポイントが明らかになった。

1. **[ポイント1]**：[説明]
2. **[ポイント2]**：[説明]
3. **[ポイント3]**：[説明]

[総括文]
```

## 専門用語の処理

### 初出時の説明

```markdown
什器（じゅうき：店舗で商品を陳列・展示するための器具）
バイヤー（小売店の仕入れ担当者）
リードタイム（発注から納品までの期間）
```

### 略語の展開

```markdown
POC（Proof of Concept：概念実証）
OEM（Original Equipment Manufacturing：相手先ブランド製造）
```

## 質問がスキップされた場合

インタビュー中に回答されなかった質問は、その旨を明記して省略可。

```markdown
### 質問6：[質問文]

※ インタビュー中、時間の都合により回答なし
```

または、部分的な回答があった場合：

```markdown
### 質問6：[質問文]

- **[得られた情報のみで構成した核心文]**
    - （詳細な回答は得られなかったが）〇〇という言及があった
```

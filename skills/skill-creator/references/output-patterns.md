# 出力パターン

スキルの出力品質を安定させるためのパターン集。

## テンプレートパターン

出力のフォーマットを定義する。求められる厳密さに応じて使い分ける。

**厳密なテンプレート**（データ形式やレポート構造を固定したい場合）:

```markdown
## 議事録フォーマット

以下のテンプレートに従って出力する。

# [会議名] 議事録

## 基本情報
- 日時: [YYYY-MM-DD HH:MM]
- 参加者: [名前のリスト]

## アジェンダ
1. [議題1]
2. [議題2]

## 決定事項
- [決定内容と担当者]

## ToDo
| 担当 | 内容 | 期限 |
|------|------|------|
| [名前] | [タスク] | [日付] |
```

**柔軟なテンプレート**（構造は示すが内容に応じて調整を許す場合）:

```markdown
## 調査レポートの構成

以下を基本構成とするが、調査内容に合わせて適宜調整する。

# [調査テーマ]

## 調査の背景と目的
[なぜこの調査が必要か]

## 主要な発見事項
[発見した内容を重要度順に記載。節の粒度は内容に合わせる]

## 示唆
[発見事項からどんな行動や判断が導けるか]
```

## 入出力例パターン

出力のトーンや粒度を言葉で説明するより、実例で示したほうが正確に伝わる。

```markdown
## メール件名の生成

以下の例に倣って件名を生成する。

入力: 来週月曜に予定されていたレビュー会議を水曜に変更したい
出力: 【日程変更】レビュー会議: 月曜 → 水曜

入力: 第3四半期の売上レポートが完成した
出力: 【共有】Q3 売上レポート（確定版）

スタイル: 【カテゴリ】+ 内容の要約。補足情報は括弧で添える。
```

例を示すことで、フォーマットだけでなく判断基準や表現の粒度まで伝えられる。

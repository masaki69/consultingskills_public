# ワークフローパターン

スキル内で処理の流れを定義するためのパターン集。

## 逐次ワークフロー

複数のステップを順番に実行する場合、全体の流れを最初に示してから各ステップを詳述する。

```markdown
データ変換は以下の手順で行う。

1. 入力ファイルの検証（validate_input.py を実行）
2. スキーマのマッピング（mapping.json を編集）
3. 変換の実行（transform.py を実行）
4. 出力の検証（validate_output.py を実行）

### Step 1: 入力ファイルの検証
...

### Step 2: スキーマのマッピング
...
```

全体像を先に見せることで、各ステップの位置づけが明確になる。

## 条件分岐ワークフロー

入力や状況によって処理を切り替える場合、判断ポイントを明示する。

```markdown
1. 入力ファイルの種類を確認する
   **.csv の場合** → 「CSV取り込み」へ
   **.json の場合** → 「JSON取り込み」へ
   **その他** → ユーザーに形式を確認する

2. CSV取り込み
   [CSVに特化した手順]

3. JSON取り込み
   [JSONに特化した手順]
```

分岐の条件は読み手が迷わない粒度で書く。曖昧な条件は誤った分岐を招く。

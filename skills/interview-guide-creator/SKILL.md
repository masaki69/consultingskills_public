# Interview Guide Creator

提案書と初期調査結果をもとに、インタビューで検証・補強すべき事項を整理し、インタビューガイド（質問リスト）を作成する。

## 入出力

- **入力**: 提案書（論点・仮説が整理されたもの）、初期調査結果（デスクリサーチ等の調査報告書）
- **出力**: インタビューガイド.md（目的・対象者・設問を含む）

## ワークフロー

1. **入力ファイルの確認**
   - 提案書から論点（大論点・小論点）と初期仮説を抽出
   - 初期調査結果から検証済み事項と未検証事項を整理

2. **検証事項の整理**
   - 提案書の論点に紐づく「インタビューで検証すべき事項」を特定
   - 初期調査で埋まらなかった情報ギャップを明確化
   - 論点ごとに「何を聞けば答えが得られるか」を設計

3. **対象者ペルソナの設計**
   - 各検証事項に回答できる最適なインタビュー対象者像を設計
   - 対象者の属性（業界経験・役職・専門性）を具体化

4. **質問設計**
   - 論点→検証事項→質問の対応関係を明確に
   - オープン質問とクローズド質問をバランスよく配置
   - 深掘り用のサブ質問を用意

## 出力フォーマット

```markdown
# インタビューガイド：[プロジェクト名]

## 1. インタビューの目的

### 調査全体における位置づけ
[本インタビューが調査プロジェクト全体でどのような役割を果たすか]

### 検証すべき事項
| 論点 | 検証事項 | 初期調査での充足度 |
|------|----------|-------------------|
| [論点①] | [検証事項] | △ 概観のみ |
| ... | ... | ... |

## 2. 想定インタビュー対象者

### 対象者タイプA：[タイプ名]
- **属性**: [業界・職種・経験年数・役職等]
- **選定理由**: [なぜこの対象者が最適か]
- **想定人数**: X名
- **主に聞きたい論点**: ①②

### 対象者タイプB：[タイプ名]
...

## 3. インタビュー設問

### [対象者タイプA向け]

#### 導入（アイスブレイク・プロフィール確認）：5分
1. [自己紹介・経歴確認の質問]
2. [現在の役割・業務範囲の確認]

#### セクション1：[論点①に対応]（XX分）
**検証したいこと**: [具体的な検証事項]

3. [メイン質問]
   - サブ質問a：[深掘り用]
   - サブ質問b：[具体例を引き出す]

4. [メイン質問]
   ...

#### セクション2：[論点②に対応]（XX分）
...

#### クロージング：5分
- [追加で聞きたいことの確認]
- [他に紹介いただける方の確認]
```

## 質問設計のルール

### 質問タイプの使い分け

| タイプ | 用途 | 例 |
|--------|------|-----|
| オープン質問 | 実態・経験を引き出す | 「〜について、どのようにお考えですか？」 |
| クローズド質問 | 仮説検証 | 「〜という理解で合っていますか？」 |
| 具体化質問 | 具体例の取得 | 「具体的な事例を教えていただけますか？」 |
| 比較質問 | 差異の把握 | 「A社とB社では、どのような違いがありますか？」 |

### 質問文のルール
- **主語と述語を明確に** - 曖昧な問いかけを避ける
- **一問一答** - 複数の論点を1つの質問に詰め込まない
- **誘導を避ける** - 「〜だと思いますが」等の誘導的表現を使わない
- **専門用語は定義を添える** - 認識齟齬を防ぐ

### 時間配分の目安
- 導入（アイスブレイク）：5分
- メインパート：45〜50分（論点ごとに10〜15分）
- クロージング：5分
- **合計**：約60分

## 論点と質問の対応

インタビューガイドの品質は「論点に対して答えが得られる設計になっているか」で決まる。

**チェックポイント**:
- [ ] 全ての大論点に対応する質問セクションがあるか
- [ ] 各質問の「検証したいこと」が明記されているか
- [ ] 初期調査で不足していた情報を埋める質問があるか
- [ ] 回答から次のアクション（追加調査・意思決定）に繋げられるか

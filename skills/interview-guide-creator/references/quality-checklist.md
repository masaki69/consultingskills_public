# インタビューガイド品質チェックリスト

## 1. 目的の明確性

- [ ] 調査全体における本インタビューの位置づけが明記されている
- [ ] 各論点について「何を検証するか」が具体的に書かれている
- [ ] 初期調査との役割分担（デスクリサーチで分かること vs インタビューで聞くこと）が整理されている

## 2. 対象者設計

- [ ] 対象者タイプごとに選定理由が明記されている
- [ ] 対象者の属性（業界・職種・経験年数・役職）が具体的
- [ ] 各対象者タイプと聞きたい論点の対応が明確
- [ ] 対象者の人数と優先度が設定されている

## 3. 質問設計

### 構成
- [ ] 導入→メインパート→クロージングの流れがある
- [ ] 各セクションの時間配分が設定されている
- [ ] 論点との対応関係が明示されている

### 質問の質
- [ ] 一問一答になっている（複合質問がない）
- [ ] 誘導的な表現がない
- [ ] オープン質問とクローズド質問のバランスが取れている
- [ ] 深掘り用のサブ質問が用意されている

### 情報取得の完備性
- [ ] 全ての大論点に対応する質問がある
- [ ] 仮説検証ができる質問がある
- [ ] 具体例を引き出す質問がある
- [ ] 数値・ファクトを取得できる質問がある

## 4. 実用性

- [ ] 質問数が時間内に収まる量（60分で10〜15問程度）
- [ ] インタビュアーが読んですぐ使える明瞭さ
- [ ] 回答から次のアクションに繋げられる設計

## NGパターン

### 質問の悪い例
| NG例 | 問題点 | 改善例 |
|------|--------|--------|
| 「御社の課題と今後の展望を教えてください」 | 複合質問 | 「現在最も大きな課題は何ですか？」→「今後3年の展望を教えてください」 |
| 「〜だと思うのですが、いかがですか？」 | 誘導的 | 「〜についてどのようにお考えですか？」 |
| 「市場環境についてどう思いますか？」 | 漠然 | 「競合A社の参入後、御社の受注状況にどのような変化がありましたか？」 |
| 「DXの取り組みについて」 | 質問になっていない | 「DXの取り組みで最も成果が出た施策は何ですか？」 |

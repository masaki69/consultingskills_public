## 背景・目的

- 配下で「背景」「現状課題」「目的」「除外」を必ず分けて記載
- 「現状課題」には、既存情報の不足点を具体的に書く

```markdown
## 1. 背景・目的

### 〇〇に向け△△を揃える

メッセージ：〇〇の意思決定に使える△△を短期で提示する

* **背景**：...
* **現状課題**：既存レポート（例：矢野「〇〇」等）は市場概況中心で、**先端技術・技術課題・プレイヤー**の深掘りが不足
* **目的**：...
* **除外**：...
```

## 初期仮説

### 形式ルール

- **完備な文章**で記載（体言止め・断片的箇条書き禁止）
- 述語：「〜と考えられる」「〜と想定される」「〜可能性が高い」等
- 主語を明確に（例：「太陽HDは〜」）
- **仮説は二層**で書く：上位仮説（構造）→下位仮説（具体）
- **論点（大論点→小論点）に直接紐づく階層**で記載する

### 数字と根拠を入れる（重要）

- 仮説には**具体的な数字**を入れる（レンジ、プレミアム額、比率など）
- 数字が入れられない場合でも、**要因の因数分解**を示す
- 「〜可能性が高い」だけの羅列は禁止

**良い例**：
- 「市場の中心レンジは、PMが1,000〜1,400万円、PLが800〜1,200万円に収斂しやすい」
- 「高騰は"需給逼迫"と"上流化"が同時に進行した結果であると考えられる」
- 「PJ規模（数億→十数億）と上流比率の有無で、100〜300万円規模の差が出る可能性が高い」

**悪い例**：
- 「年収レンジは上方に動いている可能性が高い」（数字がない）
- 「乖離が生じている可能性が高い」（要因の分解がない）

### テンプレート

```markdown
## 4. 初期仮説

### 〇〇の主因は「△△×□□」で説明できる

メッセージ：要因を因数分解し、打ち手の優先順位を付けられる形に落とす

* **大論点① 仮説：** 市場の中心レンジは〇〇〜△△に収斂し、□□の有無で100〜300万円規模の差が出る可能性が高い
  * 小論点①-1 仮説：乖離の一部は、〇〇と△△の差によって生じていると考えられる
  * 小論点①-2 仮説：〇〇層は△△より高いレンジに寄る可能性が高く、まず△△のベースラインを作ることで〇〇の上振れを説明しやすくなると想定される
```

## タスク詳細

### 形式ルール

- **参照ソースを具体的に列挙**する（抽象的な「公開情報」だけで終わらない）
- **インタビュー対象の属性**を明記する（例：転職エージェント、競合OB、潜在顧客など）
- **「提示値」と「実態値」の乖離**に着目する視点を入れる

### 良い例

```markdown
* **デスクリサーチ**
  * 参照候補ソース例（Phase1で必ず当たり、整合性を取る）
    * タレントリンク：ERPコンサル年収（紹介実績ベース）
    * ワークサーチ：給与統計（自己申告/求人由来）
    * キャリアフォワード：フリーランス調査（単価の参照点として利用）
  * ベンチマーク企業群の求人票を収集し、年収レンジ/必須要件/訴求軸/勤務地・働き方を構造化
* **エキスパートインタビュー（論点①②③の裏取り）**
  * 対象：ERP特化エージェント/ヘッドハンター
  * 聴取内容：直近12〜24か月の決定年収レンジ実例、上振れ要因、採用チャネル別の勝ちパターン
  * 「提示年収」と「決定年収」の乖離がどこで生じるかを聴取
```

### 悪い例

```markdown
* デスクリサーチ：公開情報を収集し整理する
* インタビュー：エキスパートから情報を聴取する
```

## 納品物

- **経営層向け1枚絵**を標準項目として含める
- シナリオ比較表（選択肢A/B/Cの費用対効果）を含める

```markdown
## 8. 納品物

メッセージ：〇〇と△△を経営会議で即決できる資料一式として短期間で整える

* **最終報告書（PowerPoint）**
  * 〇〇のレンジ推定（年齢×役割×経験×規模）
  * 競合の〇〇×△△×□□ベンチマーク
  * 選択肢A/B/Cの**シナリオ比較（費用・リスク・獲得確度）**
* **データブック（Excel）**
  * 集計データ（企業/職種/レンジ/要件/訴求）
  * 推定の根拠サマリ（参照ソース、インタビュー件数、集計ロジック）
* **インタビューサマリ（匿名）**
  * 論点別インサイト（決定要因、辞退要因、交渉余地の実態）
* **経営層向け1枚絵**
  * 「市場実態 vs 自社提示 vs 競合」と、推奨打ち手の要点
```

## スケジュール

- **週次で分解**する（月単位の大雑把なスケジュールは禁止）
- 各週のタスクと中間共有のタイミングを明確化

```markdown
## 9. スケジュール

メッセージ：設計・収集・分析を週次で回し、中間共有を挟んで〇月末に経営判断用最終報告へ確実に着地する

* **W1（○月上旬〜中旬）**
  * キックオフ、対象定義確定、ベンチマーク企業リスト確定
  * フィジビリティ確認（対象者母数・代替定義案）
* **W2〜W3（○月中旬〜下旬）**
  * デスクリサーチ、データ収集・構造化
  * インタビュー対象者のスクリーニング開始
* **W4（○月上旬）**
  * インタビュー本実施（並行）
  * **中間共有**（暫定レンジ、論点別の一次示唆、追加深掘り方針）
* **W5〜W6（○月中旬）**
  * 追加インタビュー、推定の精緻化、競合ベンチマーク完成
  * シナリオ比較のたたき台作成
* **W7〜W8（○月下旬〜納期）**
  * 最終資料化、経営層向け1枚絵・意思決定比較表の確定
  * **納品**
```

## 見積り

- **金額×インタビュー件数×分析範囲**の3軸で差分を説明する
- 金額は具体的に記載する（「予算上限内で調整」だけで終わらない）
- インタビュー件数は**エキスパート○件＋候補者○件（計○件）**の形式で明記

| プラン | 価格（税抜） | インタビュー設計 | 主な分析範囲 | 想定アウトプット |
| --- | ---: | --- | --- | --- |
| A：ライト | 180万円 | エキスパート6件＋候補者4件（計10） | 〇〇の骨格＋主要競合の比較 | 経営層向け1枚絵＋基本レポート |
| B：スタンダード（推奨） | 250万円 | エキスパート8件＋候補者6件（計14） | 〇〇の精緻化＋競合の勝ちパターン深掘り＋シナリオ比較 | レポート一式＋比較表 |
| C：プレミアム | 320万円 | エキスパート10件＋候補者10件（計20） | セグメント別まで分解し、打ち手を具体化 | 詳細データブック＋施策ロードマップ |

**母数不足時の方針（全プラン共通で記載）**：
- 対象を「〇〇」→「△△」へ段階拡張し、〇〇プレミアムを相対比較で示す
- 「決定実績」母数が不足する場合、「提示実績」も併用し推定精度を確保

## 支援実績の伝え方

- 3者面談前に先方へ共有する**匿名化した実績サマリ**を準備
- 類似案件の経験、アウトプットの型を示し、信頼を構築

```markdown
## 0. 支援実績の伝え方（匿名化）

メッセージ：〇〇と△△を組み合わせ、意思決定資料まで一気通貫で作る

* **先方共有用の実績サマリ文案（固有名詞・数値は伏せる前提）**
  * 〇〇領域で、△△を調査し、経営層決裁用の根拠資料に落とし込む支援を実施
  * 〇〇の定性情報と、△△の定量集計を突合し、「提示値」ではなく「実態値」推定まで踏み込む設計を採用
  * 母数不足リスクに備え、対象定義の段階的拡張と、結論の精度を落とさない代替案を準備
* **アウトプットの型（過去事例で多用する型）**
  * 「〇〇×△△×□□」の三点セットで、選択肢A/Bを比較可能にする
  * チャネル別の勝ちパターンと、必要投資（工数・費用）を並列表にする
```

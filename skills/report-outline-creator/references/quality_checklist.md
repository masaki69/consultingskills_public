# 報告書骨子 品質チェックリスト

報告書骨子の品質を確認するためのチェックリスト。

---

## 1. 構造の妥当性

### 全体構成

- [ ] **エグゼクティブサマリーが冒頭にある**: 経営層が1分で要点を把握できる構成
- [ ] **本編が論点に対応している**: 大論点の数と章の数が概ね一致
- [ ] **まとめ章がある**: 調査結果の総括と次のアクションを明示
- [ ] **Appendixが設計されている**: 調査方法、ソース、用語集の配置

### 章間のバランス

- [ ] **スライド配分が妥当**: 重要論点に十分なスライド数を割り当て
- [ ] **深掘り章と俯瞰章のバランス**: 詳細分析と全体像の両方をカバー
- [ ] **章の粒度が揃っている**: 章ごとのスライド数が極端にばらつかない（目安：各章4〜12スライド）

---

## 2. 論点の網羅性

### 論点-章対応

- [ ] **全ての大論点に対応する章がある**: 論点-章対応表で漏れがないか確認
- [ ] **小論点がサブセクションに反映されている**: 小論点①-1、①-2等が章内に配置
- [ ] **横断論点の扱いが明確**: 複数論点にまたがる内容は「示唆」セクションで統合

### 検証事項の反映

- [ ] **インタビューガイドの検証事項が反映されている**: 検証すべき事項が章構成に含まれる
- [ ] **初期調査の発見事項が反映されている**: デスクリサーチで得た示唆が章に配置

---

## 3. 想定アウトプットの具体性

### 図表・一覧

- [ ] **各章に想定アウトプットが明記されている**: 「想定アウトプット」セクションが全章にある
- [ ] **具体的な図表名が記載されている**: 「比較表」ではなく「○○比較マトリクス（軸：△△×□□）」
- [ ] **一覧・リストの内容が明確**: 「企業一覧」ではなく「VPPプレイヤー一覧（企業名/事業規模/主要サービス/強み）」

### 図表の種類（プロジェクト種類別）

**技術動向調査**:
- [ ] 技術マップ or 技術分類図
- [ ] 技術比較表
- [ ] プレイヤー一覧
- [ ] 技術ロードマップ

**市場調査**:
- [ ] 市場規模推移グラフ
- [ ] セグメント構成図
- [ ] 競合ポジショニングマップ
- [ ] シェア推移表

**インタビュー中心**:
- [ ] ビジネスモデル比較表
- [ ] 収益構造図
- [ ] エコシステム構造図
- [ ] 戦略オプション比較表

**DD**:
- [ ] 事業構成図
- [ ] 財務サマリー表
- [ ] リスクマトリクス
- [ ] バリュエーションサマリー

---

## 4. 会議転用の設計

### 中間報告

- [ ] **中間報告の構成が明記されている**: 納品時期、スライド数、内容
- [ ] **抜粋する章が具体的**: 「主要章の抜粋」ではなく「第1章から4〜5スライド」等
- [ ] **完成度の目安が明記**: 「6〜7割完成版」等の表現

### 補足資料

- [ ] **補足資料の位置づけが明確**: インタビューサマリー、詳細データ等の役割
- [ ] **補足資料の構成が設計されている**: 本文と補足の役割分担

---

## 5. 品質バー（最低ライン）

### NG項目（これがあると不合格）

- [ ] **論点-章対応表がない**: 必ず報告書末尾に配置
- [ ] **エグゼクティブサマリーがない**: 必ず冒頭に配置
- [ ] **想定アウトプットが空欄**: 各章に最低1つは明記
- [ ] **中間報告の設計がない**: 納品スケジュールに中間報告がある場合は必須
- [ ] **スライド数の目安がない**: 各章に「X〜Yスライド」の記載が必要

### 推奨項目（あると品質が上がる）

- [ ] **各章末に「示唆」セクションがある**: 「だから何？」を明確に
- [ ] **想定アウトプットに軸・項目が明記されている**: 比較表の比較軸等
- [ ] **Appendixの詳細が設計されている**: 用語集、参考文献リストの項目

---

## チェック手順

1. **構造チェック**: 全体構成→章間バランス→スライド配分
2. **網羅性チェック**: 論点-章対応表を作成し漏れを確認
3. **具体性チェック**: 想定アウトプットの図表名・軸を確認
4. **会議転用チェック**: 中間報告・補足資料の設計を確認
5. **品質バーチェック**: NG項目がないか最終確認

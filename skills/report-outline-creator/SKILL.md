# 報告書骨子作成スキル

提案書の論点構造と調査結果をもとに、最終報告書の骨子（章立て・スライド構成・想定アウトプット）を設計する。

## 入出力

- **入力**: 提案書（論点・仮説）、インタビューガイド（検証事項）、初期調査結果
- **出力**: 報告書骨子.md（章立て・スライド構成・想定アウトプット）

## ワークフロー

1. **入力ファイルの確認**
   - 提案書から論点（大論点・小論点）と初期仮説を抽出
   - 納品物定義（PPT章立て、一覧等）を確認
   - インタビューガイドから検証事項と対象者タイプを確認

2. **プロジェクト種類の判定**
   - 入力から調査の性質を判定（技術動向/市場調査/インタビュー中心/DD）
   - `references/structure_templates.md` から該当テンプレートを参照

3. **章構成の設計**
   - 論点を章にマッピング（論点-章対応ルール参照）
   - 各章のスライド数と構成要素を設計
   - エグゼクティブサマリー、本編、まとめ、Appendixの流れを確保

4. **想定アウトプットの具体化**
   - 各章で必要な図表・比較マトリクス・一覧を明示
   - 会議転用（中間報告）の設計

5. **品質チェック**
   - `references/quality_checklist.md` に基づき品質を確認

## 論点-章対応ルール

### 基本原則

| 論点の性質 | 対応する章の種類 |
|-----------|----------------|
| ビジネスモデル・収益構造 | 企業分析章（深掘り分析） |
| 市場規模・成長性 | 市場動向章 |
| 競合・ポジショニング | 市場動向章または競合分析章 |
| 技術・製品特性 | 技術分析章 |
| 提供価値・サービス設計 | サービス/エコシステム章 |
| 戦略オプション・参入方法 | 戦略オプション章 |

### マッピング手順

1. **大論点→章タイトル**: 大論点1つに対し、原則1つの章を対応させる
2. **小論点→サブセクション**: 小論点は章内のサブセクション（1-1, 1-2等）に対応
3. **横断論点の扱い**: 複数論点にまたがる内容は「示唆」セクションで統合

### 例

```
大論点① ビジネスモデル・契約実態
  └→ 第1章：VPP主軸プレーヤーのビジネスモデル
      ├ 1-1. 市場構造整理（小論点①-1）
      ├ 1-2. 企業A分析（小論点①-2）
      ├ 1-3. 企業B分析（小論点①-2）
      └ 1-4. 比較と示唆（統合）
```

## 出力フォーマット

```markdown
# 最終報告書 骨子案

## [クライアント名] 御中
## [プロジェクト名] 最終報告書

---

## ドキュメント構成

| ドキュメント | 形式 | 納品時期 | 用途 |
|-------------|------|----------|------|
| **最終報告書（本書）** | PowerPoint | [日付] | 社内報告・意思決定資料 |
| **中間報告資料** | PowerPoint | [日付] | 社内会議向け速報 |
| **[補足資料名]** | Word/PDF | [日付] | 詳細エビデンス・参照資料 |

---

## 最終報告書 章立て

### 表紙・目次（2スライド）

- 表紙：プロジェクト名、クライアント名、提出日、提出者
- 目次：章構成と参照ページ

---

### エグゼクティブサマリー（3〜4スライド）

**目的**: 経営層向けに調査結果の要点を1分で伝える

| スライド | 内容 |
|----------|------|
| **ES-1** | 調査の背景・目的・調査範囲の確認 |
| **ES-2** | 主要発見事項（Key Findings）3〜5点のサマリー |
| **ES-3** | 戦略オプションの全体像と推奨の方向性 |
| **ES-4** | 次のアクション（検討すべき論点・追加調査の要否） |

---

### 第N章：[章タイトル]（X〜Yスライド）

**対応論点**: 大論点N（[論点名]）

#### N-1. [サブセクションタイトル]（Zスライド）
- [内容項目1]
- [内容項目2]

#### N-2. [サブセクションタイトル]（Zスライド）
...

#### N-X. [章名]からの示唆（1スライド）
- [示唆内容]

**想定アウトプット**:
- [図表名1]
- [比較マトリクス名]
- [一覧名]

---

### 第X章：まとめと次のステップ（2スライド）

#### X-1. 調査結果のまとめ
- 各章の要点を1スライドで再整理

#### X-2. 次のアクション提言
- 短期：[アクション内容]
- 中期：[アクション内容]

---

### Appendix（必要に応じて）

- A-1. 調査方法・情報ソース一覧
- A-2. インタビュー実施概要（対象者属性・実施日・所要時間）
- A-3. 用語集
- A-4. 参考資料・引用文献

---

## 中間報告資料の構成

**目的**: 社内会議向けに、X割完成版として主要発見事項を速報

| セクション | スライド数 | 内容 |
|-----------|-----------|------|
| 表紙・目次 | 1 | |
| エグゼクティブサマリー | 2 | 主要発見事項のドラフト |
| [章名]抜粋 | X〜Y | [内容] |
| 次のステップ | 1 | 残り調査項目・方向性確認 |
| **合計** | **XX** | |

---

## 論点と報告書章の対応表

| 大論点 | 小論点 | 対応章 |
|--------|--------|--------|
| ① [論点名] | ①-1 [小論点名] | 第N章 |
| | ①-2 [小論点名] | 第N章 |
| ② [論点名] | ②-1 [小論点名] | 第M章 |
...
```

## プロジェクト種類別ガイド

プロジェクトの性質に応じて、重点を置く章と構成が異なる。詳細は `references/structure_templates.md` を参照。

| 種類 | 重点章 | 特徴 |
|------|--------|------|
| **技術動向調査** | 技術マップ、プレイヤー分析 | 技術比較表、ロードマップが必須 |
| **市場調査** | 市場規模、競合分析 | 市場規模推移グラフ、シェア図が必須 |
| **インタビュー中心** | ビジネスモデル、戦略オプション | 企業深掘り分析、インタビューサマリーが必須 |
| **DD** | 事業概要、リスク評価 | 財務分析、バリュエーションが必須 |
